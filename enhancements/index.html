All Moves As First (AMAF)

All Moves As First (AMAF) is a method commonly used in earlier Monte Carlo Go programmes to accelerate node reward estimate reliability. The basic principle is to update the reward value and visit count for not only the current move but for all further moves (by the same player) further down current playout sequence, the rationale being that the final result of most move sequences will be the same regardless of the order in which those moves were made.

AMAF is a fundamental domain-independent enhancement that can yield significant performance benefits. It is usually one of the first enhancements to try for any MCTS application.

Basic AMAF
The basic AMAF method adds the AMAF update values directly to existing UCT values, and these combined values are used for UCB node selection as usual. This is the simplest AMAF approach but can actually prove detrimental, as UCT reward values are combined with less reliable AMAF reward values even after nodes have been visited a sufficient number of times for the UCT estimates to be reliable.

See: Helmbold and Parker-Wood (2006).
